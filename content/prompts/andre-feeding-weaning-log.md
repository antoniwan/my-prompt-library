# Andre Feeding & Weaning Log

Reusable procedure for logging a feeding. Paste this, or say "log Andre's [food]" and follow it. Turns a repeated one-off into a fixed, auditable routine.

## Trigger

Any request to record a feeding, a first-food try, an allergen result, or a milk/formula change for Andre.

## Canonical target (one file only)

Log to `Learning/Fatherhood/Baby-Led Weaning Guide - Andre.md`.

- `Active Projects/First Bites - Andre Field Guide.md` is reference only (prep rules, allergen protocol, choking distinction). Do not log food there.
- `Learning/Fatherhood/Andre - Weaning Guide.md` is superseded (2026-07-22) and a candidate for `_archive/`. Do not log there.

If a request would write a feeding anywhere else, stop and redirect to the canonical file.

## Two hard rules

1. Log only what actually happened. Never pre-log a plan, an intention, or "tomorrow's" food. If it has not been eaten, it does not get a line. This is the say-do rule from [[Learning/Self-Understanding/Say-Do Coherence as Core Wound]]: the log records the *do*, not the intent.
2. One new thing at a time. Introduce a single new food or allergen per sitting so any reaction has one clear cause. If two new items were served together, note that the reaction read is not clean.

## Where inside the file

- First try of a food: go to Part 7 (Food Tables by Month), find the table for Andre's current stage, and fill the `Date tried`, `Reaction`, and `Notes` cells in that food's row. Add a new row only if the food is absent from the table.
- Allergen closed: also mark the Part 0 "Allergen closure schedule" table, `Done / reaction` column, with the date and result.
- Milk or formula change: append to the "Milk Transition" Log section.

## Fields to capture every time

- Date, and time when it matters (early morning iron, late dinner).
- Food, and form or prep including boil/steam method and time (example: "steamed, rolling boil ~16 min").
- Reaction. For any allergen, note the 24 to 48 hour delayed-response watch.
- Notes: amount, texture, cleanup observation, whether it was the iron anchor for the day.

## After logging

- Bump `updated:` in the guide's frontmatter to today's date.
- Iron check: iron is the daily non-negotiable (Part 3). If the day's log has no iron food, flag it.
- Report back one line: date, food, prep, reaction, and which table you wrote to.

## Output format

Match the existing table columns exactly. Do not restructure the tables. One filled or added row per food, nothing more.
