# MPL --- Expectations Document

## 1. Purpose

MPL exists to:

-   Demonstrate structured AI thinking
-   Curate high-quality prompts
-   Promote sovereign AI workflows
-   Provide reusable, performance-oriented prompt assets

This is not a prompt dumping ground.

------------------------------------------------------------------------

## 2. Philosophy

### 2.1 Sovereignty

Prompts are: - Markdown - Version-controlled - Portable - Owned by the
repository

No lock-in. The **imports** workflow lets users bring prompts from other tools (e.g. Obsidian, Notion) into the repository format; see [REQUIREMENTS ยง1.8](REQUIREMENTS.md#18-imports--migration) and [SPEC ยง9](SPECIFICATIONS.md#9-imports-and-migration).

### 2.2 Curation Over Quantity

Quality \> volume. Every prompt must: - Be structured - Have clear
intent - Be tested - Provide measurable value

### 2.3 Engineering Discipline

This is not a Notion page. This is not a random collection. This is a
system.

------------------------------------------------------------------------

## 3. What This Is Not

-   Not a SaaS (v1)
-   Not an AI wrapper tool
-   Not a chatbot interface
-   Not a marketplace

It is a curated knowledge system.

------------------------------------------------------------------------

## 4. Success Criteria

### 4.1 Short-Term

-   20--30 high-quality prompts
-   Clean architecture
-   Clear tagging taxonomy
-   Strong Lighthouse score
-   Clear Git contribution workflow

### 4.2 Medium-Term

-   Community contributions
-   Recognized as structured prompt reference
-   Consistent updates
-   SEO traction

### 4.3 Long-Term (Optional)

-   Integration with tooling
-   Programmatic access
-   Reputation-building asset
-   Foundation for AI consulting or productization

------------------------------------------------------------------------

## 5. Personal Expectation

Build this: - Cleanly - Intentionally - Without overengineering -
Without ego - Without rushing

It should feel engineered. Not improvised.
